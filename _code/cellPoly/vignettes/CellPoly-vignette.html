Overview

The CellPoly package is designed to facilitate plotting cells polygonal boundaries while working in an R session. For each cell, it finds a polygon encompassing its transcripts. This will diverge from segmentation masks, but it will place each transcript in the same cell (ignoring 3-d considerations).

The only necessary input is a data frame of transcript positions, along with the cell IDs they are assigned to.

This vignette demonstrates the use of the CellPoly package to plot CosMx data. CosMx is a spatial molecular imager for sub-cellular resolution of mRNA transcripts and protein targets. We analyze a subset of a lupus nephritis experiment.

Package architecture & workflow:

CellPoly works on a named list of polygons, which each element corresponding to a unique cell ID. To begin, the elements of this list are empty. Cell polygons can be calculated up-front using the cellPolys() function, or they can be computed on the fly while drawing plots using the drawPolys() function. The workflow can be arranged such that only the polygons needed for a given plot are computed, and so no cells polygon need be computed twice. As code{cellPolys} or (which itself calls ) is run, any polygons they compute get added to the ever-growing polygons list.

Now we can plot polygons:

drawPolys(polys, 
          cell_ids = NULL, #tempids, 
          col = scales::alpha(cosmx_nephritis$cellcols[cosmx_nephritis$celltype[names(polys)]], 0.5),
          border = cosmx_nephritis$cellcols[cosmx_nephritis$celltype[names(polys)]], 
          add = FALSE, outputpolys = FALSE, xaxt = "n", yaxt = "n", xlab = "", ylab = "", asp = 1)

The assignment back to polys is important, since drawPolys returns the polygons list. (This is because it can add to the polygons list if it was incomplete to begin with.)

To finish this example, you might want to plot a couple genes across this view:

drawPolys(polys,

# which genes to show and their colors:
genecols = c("IFITM1" = "red", 
             "IFITM3" = "blue") 
# points for genes:
for (gene in names(genecols)) {
  points(transcript_df[transcript_df$target == gene, c("x", "y")], 
         col = genecols[gene], 
         pch = 16, cex = 0.2)
}
legend("bottomleft", col = genecols, legend = names(genecols), pch = 16, cex = 0.9)

Inattentive workflow

You can also work with just the drawPolys() function, skipping the cellPolys call:

#initialize:
polys <- initPolys(cell_ids = unique(transcript_df$cell_id))

# draw a few polygons, saving them so they don't need to be recalcualted
polys <- drawPolys(polys, 
                   cell_ids = names(polys)[c(1,200, 400, 700, 900)], 
                   transcript_df = transcript_df,
                   col = "#00008B80", border = "#00008BFF", 
                   add = FALSE, 
                   outputpolys = TRUE,
                   type = "chull",
                   xaxt = "n", yaxt = "n", xlab = "", ylab = "", asp = 1)
